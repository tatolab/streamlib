# camera-rust-plugin

Camera → a **local Rust cdylib plugin** (grayscale) → Display. Shows how an app
ships its own processor as an in-repo package.

## The model this example teaches

The sibling `plugin/` crate is a cdylib package (`@tatolab/camera-rust-plugin`)
carrying the `GrayscaleRust` processor. App code references it exactly like any
other processor — version-free, no load call:

```rust
let grayscale = runtime.add_processor(ProcessorSpec::new(
    processor_type_ref!("tatolab", "camera-rust-plugin", "GrayscaleRust"),
    serde_json::Value::Null,   // config-less processor → Null, not json!({})
))?;
```

`./setup.sh` links `./plugin` into `streamlib_modules/` alongside
`@tatolab/camera` and `@tatolab/display`; the runtime lazily discovers, builds,
and loads the plugin on first reference. There is no manual cdylib staging and
no `add_module` call.

## Run it

```bash
./setup.sh    # link SDK + camera/display + ./plugin into streamlib_modules/
cargo run
```

`./setup.sh` runs `streamlib link --engine <checkout>` (Rust SDK), records
`STREAMLIB_LINK_CHECKOUT` in `.cargo/config.toml` `[env]` (so the plugin's
build-time schema codegen resolves `@tatolab/core` from the checkout), and
`streamlib link` for camera, display, and `./plugin`. The `streamlib` CLI must
be on your `PATH` (`cargo build -p streamlib-cli`).

## Layout

- `src/main.rs` — the app (loader).
- `plugin/` — the `grayscale-plugin` cdylib package: `streamlib.yaml` (declares
  `@tatolab/camera-rust-plugin` + `GrayscaleRust`), `build.rs` (JTD codegen),
  `src/` (the processor). A workspace member so `cargo build` compiles it.

`streamlib_modules/`, `streamlib.lock`, `Cargo.lock`, and the
`streamlib link --engine` override are gitignored (regenerated by `./setup.sh`).
