# microphone-reverb-speaker

Microphone → CLAP reverb → speaker, with CLAP as the "shader language for
audio". **Deferred no-op at HEAD.**

Running the binary prints a deferral message and exits. The CLAP host
(`@tatolab/clap`) is macOS/iOS-only, and the pipeline relies on `ClapScanner`
(a compile-time plugin-discovery API) plus the deprecated typed-struct
processor API — there is no runtime graph-API equivalent for CLAP plugin
discovery yet. The real pipeline lives in git history; `src/main.rs` carries
the full deferral note and the path back.

## The model this example will teach once restored

App code never loads modules. The whole pipeline is `add_processor` + `connect`
+ `start`, referencing each processor with the **version-free**
`processor_type_ref!("org", "package", "Type")` macro — no `add_module`, no
registry, and no version anywhere in `src/main.rs`. Each referenced processor's
package lives in this app's **`streamlib_modules/`** folder (the streamlib
equivalent of `node_modules/`), populated by `./setup.sh`, and the runtime
lazily loads it on first reference.

## Local dev

```bash
./setup.sh        # links the in-repo SDK (this no-op loads no packages)
cargo run         # prints the deferral message and exits
```

`./setup.sh` runs `streamlib link --engine <checkout>` so the app's
`streamlib = "0.6"` dependency resolves against the in-repo SDK (a transient
`[patch.crates-io]`; there is no hosted registry — the linked checkout is the
SDK source). The `streamlib`
CLI must be on your `PATH` (build it with `cargo build -p streamlib-cli` from
the checkout); `setup.sh` falls back to the checkout's built binary otherwise.

`streamlib_modules/`, `streamlib.lock`, and `Cargo.lock` are **not committed** —
they are regenerated by `./setup.sh`, exactly like `node_modules/`.
