# nvpro-vulkan-video Test Suite

## Quick Start

```bash
# Generate fixture files (requires ffmpeg)
./tests/generate_fixtures.sh

# Run all tests
./tests/run_tests.sh

# Run specific test categories
./tests/run_tests.sh --unit       # Unit tests only (no GPU)
./tests/run_tests.sh --gpu        # GPU tests only
./tests/run_tests.sh --encode     # Encode tests only
./tests/run_tests.sh --decode     # Decode tests only
./tests/run_tests.sh --pipeline   # Pipeline round-trip only
```

## Test Binaries

### `encode-test` — Encoder Tests

No-GPU mode (default): 115 tests covering GOP structure, codec configs
(H.264/H.265/AV1), PSNR computation, DPB management, and parameter set
generation.

GPU mode: Encodes 30 frames of SMPTE color bars via Vulkan Video,
validates with ffprobe, compares source vs decoded PSNR.

```bash
cargo run --bin encode-test              # 115 config tests (no GPU)
cargo run --bin encode-test -- --gpu     # GPU encode test
```

### `decode-test` — Decoder/Parser Tests

No-GPU mode (default): Bitstream parser tests for H.264, H.265, AV1
(byte stream parsing, NAL unit extraction, scaling lists, Exp-Golomb,
SPS/PPS parsing).

GPU mode: Decodes an H.264 IDR frame using the Vulkan Video decoder,
reads back pixels, saves to PNG.

```bash
cargo run --bin decode-test              # Parser tests (no GPU)
cargo run --bin decode-test -- --gpu     # GPU decode test
```

### `pipeline-test` — Full Pipeline (Encode -> Decode)

Multi-codec test matrix that probes hardware capabilities and runs
encode/decode tests for all supported codecs. Skips gracefully when
hardware doesn't support a codec.

```bash
cargo run --bin pipeline-test
```

**Codec matrix** (RTX 3090):

| Codec | Encode | Decode |
|-------|--------|--------|
| H.264 | PASS   | Ready  |
| H.265 | Known issue (configure) | Ready |
| AV1   | N/A (not supported) | Ready |
| VP9   | N/A (not in ash 0.38) | N/A |

**Verification methods** (per encode test):

1. **Screenshot comparison** -- Extracts frame 0 from the source NV12
   fixture and from the encoded output as PNGs via ffmpeg. Saved to
   `/tmp/nvpro_pipeline_debug/` with names like
   `h264_source_frame00.png`, `h264_encoded_frame00.png`,
   `h265_source_frame00.png`, `h265_encoded_frame00.png`. Decode tests
   also produce `h264_decoded_frame00.png`.

2. **Metadata validation via ffprobe** -- Runs `ffprobe -show_entries
   stream=codec_name,profile,width,height,nb_read_frames` on each
   encoded file and checks:
   - `codec_name` matches expected (`h264` or `hevc`)
   - `profile` matches expected (`High` for H.264, `Main` for H.265)
   - `width` / `height` match configured dimensions (640x480)
   - `nb_read_frames` matches total frames encoded
   Each check prints `[PASS]` or `[FAIL]`.

3. **PSNR via ffmpeg** -- Computes PSNR between the raw NV12 source
   fixture and the encoded output using the `psnr` lavfi filter:
   `ffmpeg -f rawvideo -pix_fmt nv12 -s 640x480 -i source.yuv
   -i encoded.h264 -lavfi psnr -f null -`. Parses and prints the
   average PSNR value.

4. **SSIM via ffmpeg** -- Same approach as PSNR but with the `ssim`
   lavfi filter. Reports the `All:` average SSIM value.

5. **Summary table** -- At the end, prints a table showing all codec
   tests with their status, PSNR, SSIM, frame count, and output paths.
   Also lists all screenshot/output files in the debug directory.

## Fixtures

Fixtures are generated via `./tests/generate_fixtures.sh` and stored in
`tests/fixtures/`. They are `.gitignore`d since they're large binary files.

All encoded fixtures are generated by ffmpeg **software encoders** (libx264,
libx265) — never hardware/Vulkan encoders. This ensures decode tests use
known-good input that is independent of our own encoder.

| File | Description |
|------|-------------|
| `testsrc2_640x480_nv12.yuv` | 300 frames (10s) of testsrc2, raw NV12 — encoder input |
| `testsrc2_640x480_bgra.raw` | 300 frames (10s) of testsrc2, raw BGRA — RGBA encode path |
| `testsrc2_640x480.h264` | H.264 bitstream (libx264, IPP, High profile) — decoder input |
| `testsrc2_640x480.h265` | H.265 bitstream (libx265, IPP, Main profile) — decoder input |

Requires ffmpeg with libx264/libx265. System ffmpeg on Ubuntu may lack these;
use a static build: `FFMPEG=/path/to/ffmpeg ./tests/generate_fixtures.sh`

To regenerate: `FFMPEG=/path/to/ffmpeg ./tests/generate_fixtures.sh`
To clean: `./tests/generate_fixtures.sh --clean`

## Hardware Requirements

- **Minimum**: Any GPU with Vulkan 1.3 and video queue extensions
- **Recommended**: NVIDIA RTX 3090 or newer (supports H.264/H.265 encode + H.264/H.265/AV1 decode)
- **AV1 Encode**: RTX 4000 series or newer
- **VP9**: Not yet exposed in ash 0.38

The test suite probes hardware capabilities at runtime and skips
unsupported codecs with `[SKIP]` rather than failing.

## Unit Tests

563 unit tests run via `cargo test`. These cover all ported modules:

- Bitstream parsers (H.264, H.265, AV1)
- Exp-Golomb encoding/decoding
- Scaling lists
- GOP structure
- DPB management (H.264, H.265, AV1)
- Encoder config (profiles, levels, rate control)
- PSNR computation
- Buffer/image resource management

## Known Issues

1. **H.265 Encode**: `configure()` returns `ERROR_INITIALIZATION_FAILED`.
   The video session creation needs H.265-specific tuning. H.264 encode
   works correctly.

2. **Decoder DPB Readback**: The Vulkan decoder accepts bitstream without
   errors but the DPB readback returns zeroed data. The encode side is
   verified correct via ffmpeg decode (41.9 dB PSNR). The decoder-side
   SPS/PPS setup needs investigation.

## Agent Workflow

To run the full test workflow as an agent:

```bash
# 1. Build everything
cargo build

# 2. Run unit tests
cargo test

# 3. Generate fixtures
./tests/generate_fixtures.sh

# 4. Run encode tests (no GPU)
cargo run --bin encode-test

# 5. Run decode/parser tests (no GPU)
cargo run --bin decode-test

# 6. Run GPU encode test
cargo run --bin encode-test -- --gpu

# 7. Run full pipeline test (all codecs)
cargo run --bin pipeline-test

# 8. Check output files
ls -la /tmp/nvpro_pipeline_debug/
ffprobe /tmp/nvpro_pipeline_debug/encode_test.h264
```

Or use the all-in-one runner:

```bash
./tests/run_tests.sh
```
